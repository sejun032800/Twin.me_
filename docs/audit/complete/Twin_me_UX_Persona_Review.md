## 1. 온보딩 플로우 체험기

### 1-1. 웰컴 화면 — `app/(auth)/welcome.tsx`

**첫인상은 좋다.** 다크 배경에 `Twin.me` 로고가 코랄 컬러로 박힌 웰컴 화면은 분위기 있음. "나를 닮은 AI와 함께 연애를 돌아보세요"라는 카피는 짧고 정확하게 핵심을 찌름.

**그런데** "나중에 둘러볼게요" 버튼을 눌렀을 때 탭 화면으로 바로 진입되는 게스트 모드가 있는데, 이 상태에서 앱을 만지면 데이터가 하나도 없어서 모든 화면이 비어 보임. 게스트 유저에게는 "둘러보기용 샘플 데이터"가 있어야 함.

```
느낀 점 (하준): 디자인은 Toss 느낌인데, 로그인 강제 유도가 아닌
게스트 모드를 열어준 건 좋음. 그냥 둘러보다 삭제할 것 같긴 하지만.
```

---

### 1-2. 프로필 입력 — `app/(auth)/profile.tsx`

이름과 MBTI를 고르는 화면. MBTI 4축을 버튼으로 선택하는 방식은 직관적이고 깔끔함. **연애 시작일 입력은 `YYYY-MM-DD` 형식을 타이핑해야 한다**는 게 2025년 앱에서 보기 드문 마찰. DatePicker UI가 없고 텍스트 입력이라서 서연이 처음에 `2024.01.15`라고 입력했다가 오류 alert을 받게 됨.

**개선 포인트:**
- 연애 시작일 → DatePicker 또는 달력 슬라이더로 교체
- "우리 기념일은 언제야?" 같은 감성 언어로 필드 레이블 변경

---

### 1-3. 카카오톡 업로드 — `app/(auth)/kakao-upload.tsx`

**이 단계가 온보딩 최대 이탈 포인트다.**

흐름을 보면: DocumentPicker로 `.zip` 또는 `.txt` 선택 → JSZip으로 압축 해제 → txt 파싱 → D0 분석 → 제네시스로 이동. 기술적으로는 깔끔하게 구현돼 있음. 문제는 **유저에게 이 화면이 갑자기 나온다**는 것.

서연은 카카오톡에서 대화를 내보내는 방법을 모름.  
하준은 방법을 알지만 zip 파일이 어디 저장됐는지 못 찾음.  
둘 다 이 단계에서 10분 이상 헤맬 가능성이 높음.

코드에 `handleSkip()`이 있어서 건너뛸 수 있다는 건 구원임. 그런데 건너뛰면 트윈 AI의 말투 복제 품질이 낮아지는데, 그 트레이드오프를 유저에게 설명하는 카피가 없음.

**개선 포인트:**
- 업로드 화면 진입 전에 "30초 가이드" 화면 삽입
  - 카카오톡 → 채팅방 → 우상단 메뉴 → 대화내보내기 → 내 기기 저장 순서를 GIF 또는 스크린샷 3장으로
- 건너뛰기 버튼 카피를 "건너뛰기 (트윈 말투 정확도 낮아짐)"으로 명확화
- 업로드 실패 시 "다시 시도" + "그냥 넘어가기" 두 경로 제공
- `.zip` 파일 안에 `.txt`가 없을 때 발생하는 alert("파일 없음") 이후 복구 UX 필요

---

### 1-4. 제네시스 인터뷰 — `app/(auth)/genesis.tsx`

**이 화면이 앱 전체에서 가장 강력한 경험이다.**

4막 구조로 진행되는 에니어그램 인터뷰는 단순한 설문이 아님. "연인이 약속을 자주 어기면 화보다 먼저 드는 감정이 뭐야?", "가질 수 없거나 멀리 있는 것에 더 끌린 적 있어?" 같은 질문들은 카카오 심리테스트와 차원이 다른 깊이가 있음. 베이지안 확률이 수렴하면서 클레이 아바타가 점점 또렷해지는 구조는 직접 보이지 않더라도 "내 색깔이 만들어지는 느낌"을 줌.

```
느낀 점 (서연): 이 질문들 진짜 신기하다. 나 4유형인 것 같은데
하준도 해봐야 할 것 같아. 스크린샷 찍어서 보내야지.
```

**제네시스 완료 직후가 최초의 감성 피크**다. 오라 색과 에니어그램 유형이 확정되는 순간 → 이걸 파트너에게 공유하고 싶어지는 욕구가 자연스럽게 발생함. 그런데 현재 코드에는 이 순간에 명시적인 "파트너에게 보내기" CTA가 없음. 바로 탭으로 넘어가버림. **이 찰나를 놓치는 게 가장 큰 바이럴 미스**.

**개선 포인트:**
- 제네시스 완료 → 오라 리빌 애니메이션 화면 추가
- "내 오라 색 공유하기" + "파트너도 해보게 초대하기" CTA를 이 순간에 삽입
- 공유 카드에 오라 색과 에니어그램 유형 + 방향 언어("단단하게 곁을 지키는 타입") 포함

## 2. 메인 탭 체험기

### 2-1. 홈 탭 — `app/(tabs)/index.tsx`

**정보 과부하.**

첫 실행 시 홈 화면에서 유저가 마주하는 것들:
1. `PartnerStatusBar` — 파트너 미연동 상태라 초대 유도 배너
2. `OverflowBanner` — 이게 뭔지 맥락 없이 등장
3. `ClayTwinAvatar` — 귀엽긴 한데 탭 할 수 있는 게 없음
4. `AICoachingCard` — 데이터가 없는 상태라 내용이 빈약
5. `CircularGauge` + 숫자 점수 — **이 숫자가 뭔지 아무 설명 없음**
6. 티어 텍스트 + 이모지
7. 상태 메시지 ("오늘도 잘 하고 있어요")
8. 무드 태그 3개
9. 트윈과 대화 / 히스토리 보기 버튼
10. `MemoryRingSection`
11. 공유하기 버튼

이걸 스크롤 한 번에 다 소화하는 건 불가능함. Toss가 잔액 하나만 보여주는 이유가 있음.

`S_Current`, `S_Live`, `S_Base` 세 가지 점수 체계가 내부적으로 존재하는데, 유저 입장에서는 "이 숫자가 왜 저번이랑 다르지?"라는 혼란만 생김. 점수가 변동하는 이유를 스토리로 연결하는 장치가 없음.

**개선 포인트:**
- 홈 화면 최상단: 오라 색 + 오늘의 한 줄 상태 메시지만 표시
- 점수 게이지는 스크롤 아래로 내려서 "관심 있는 유저"만 보도록
- 첫 실행 7일간 점수 대신 "아직 만들어지는 중이에요 — 트윈과 대화할수록 정확해져요" 문구로 대체
- `OverflowBanner`는 실제로 임계값을 넘을 때만 등장하도록 (항상 노출 금지)

### 2-2. 채팅 탭 — `app/(tabs)/chat.tsx`

**구조는 좋다. 진입 장벽이 문제다.**

3룸 구조(연인방·트윈방·분석가방)가 인스타 DM 스타일 리스트로 보이는 건 익숙하고 직관적임. 특히 트윈방 컨셉 — "나를 닮은 AI와 솔직하게 대화해요" — 은 진입 욕구를 자극하는 가장 좋은 카피임.

**연인방 잠금 구조의 문제:**  
파트너 미연동 상태에서 연인방을 탭하면 🔒 플레이스홀더만 보임. 앱에서 가장 기대했던 공간이 잠겨있다는 걸 첫날 알게 되면 기대감이 꺾임. 싱글플레이어 모드에서도 "파트너가 연결되면 어떻게 쓸 수 있는지" 미리 체험할 수 있는 프리뷰 모드가 필요함.

**무료 플랜 4회 제한의 타이밍 문제:**  
트윈방·분석가방은 월 4회 무료 제한. 앱에 진심이 생기기 전에 먼저 벽이 생기는 구조. 4번 쓰고 막히는 경험은 기분 나쁨. 처음 2주간은 무제한으로 풀어주고 이후 제한하거나, 남은 횟수를 예쁘게 시각화해서 "아껴서 써야지"라는 감정으로 전환하는 접근이 필요함.

**잘 설계된 것들:**
- AI 말풍선 롱프레스 → 말투 피드백 시트 ("너무 다정했어 / 너무 딱딱했어 / 유머 안 맞아") — 이 디테일은 진짜 좋음
- `MuseSheet` (✨ 버튼) — 대화 맥락 기반 메시지 제안 3개. 서연이 "뭐라고 보내야 할지 모르겠을 때" 누를 것 같음
- `MagicMirrorModal` — 말투 분석 동의를 받는 구조 자체가 신뢰감 있음
- 민감 표현 감지 후 차단 대신 넛지로 처리하는 방식 — 압박감 없이 자연스럽게 유도함
- `CrisisOverlay` — 관계 급변 감지 시 상단 경고 오버레이. 이게 진심으로 울리면 유저 충성도 수직 상승할 것

**트윈 AI 말투 복제 품질이 전부다:**  
`toneVector` 기반으로 웃음체 빈도, 이모지 밀도, 발화 앵커를 반영하는 구조가 코드에 있음. 이게 실제로 "와 진짜 나같다"를 만들어내면 리텐션 훅이 됨. 반대로 어색하면 즉시 신뢰 붕괴. 카카오 데이터 없이 들어온 유저한테도 어느 정도 말투가 흉내내지는지가 관건.

```
느낀 점 (하준): 트윈방에서 내 말투로 대답해줬을 때 소름 돋음.
"야 이거 진짜 나같아" 하고 서연한테 보여줬음.
→ 서연 반응: "그럼 나도 해야겠다"
```

### 2-3. 히스토리 탭 — `app/(tabs)/history.tsx`

**감성 자산이 가장 많이 묻혀 있는 탭이다.**

아카이브(Helix 앨범) / 지도 / 피드 3서브탭 구조. 각각 잘 설계돼 있지만 유저가 탭을 탐색하기 전에 이미 지쳐있을 가능성이 있음 — 홈과 채팅에서 이미 많은 걸 처리했기 때문.

**잘된 것들:**

`ArchiveTab` — "✨ 우리만의 시간 / 카카오톡이 기억하는 가장 다정한 순간들" 카피는 진짜 울림이 있음. Helix 스크롤 애니메이션(parallax + scale/opacity 보간)이 제대로 렌더링되면 시각적으로 인상적일 것. `WrappedModal` — "✨ 우리의 Wrapped 보기" 버튼은 스포티파이 연말 결산처럼 공유 욕구를 자극하는 구조. 이걸 커플이 함께 여는 경험을 연례 이벤트로 포지셔닝할 수 있음.

**아쉬운 것들:**

`HELIX_CARDS`가 하드코딩된 샘플 데이터 6개. 카카오 데이터와 실제 연동되기 전까지 유저가 보는 건 픽션임. 이 사실을 유저가 눈치채는 순간 앱 신뢰도가 떨어짐. 플레이스홀더를 쓰더라도 "카카오 대화를 업로드하면 우리 실제 추억으로 채워져요" 같은 안내가 필요함.

`MapTab` — 카카오맵 API 키 없으면 플레이스홀더. 장소 추가 UI는 있지만 날짜를 `YYYY-MM-DD` 텍스트 입력으로 받는 문제가 프로필 화면과 동일하게 반복됨.

`FeedTab` — "익명의 [💛 골드] 커플"이 공유한 데이트코스 피드는 흥미로운 아이디어. 그런데 `isMockCourses` 상태일 때 "📋 샘플 데이터입니다" 배너가 뜨는데, 이게 앱이 아직 미완성이라는 인상을 직접적으로 줌. 배너 노출 방식을 좀 더 자연스럽게 처리할 필요 있음.

### 2-4. 설정 탭 — `app/(tabs)/settings.tsx`

**프라이버시 슬라이더는 신뢰 형성에 효과적이다.**

0(보호) / 1(최적화) / 2(완전복제) 3단계 슬라이더로 AI 학습 범위를 유저가 직접 제어하는 구조. "🛡️ 보호 — 최소한의 데이터만 학습해요"부터 "💗 완전복제 — 모든 대화를 학습해 나를 완벽히 복제해요"까지 카피가 솔직하고 명확함. 개인정보에 민감한 2030 유저에게 통제감을 주는 좋은 UX.

테마 설정(라이트/다크/6시그마 테마)도 잘 구현돼 있음. 특히 6시그마 테마는 앱의 차별점이 될 수 있음 — 이게 뭔지 모르는 유저를 위한 미리보기 기능이 있으면 좋겠음.

파트너 초대 코드 생성(`generateInviteCode()`)이 설정 탭 안에 묻혀 있는 건 아쉬움. 파트너 연동은 앱의 핵심 행동인데, CTA 위계상 더 눈에 잘 띄어야 함.

| 카카오 zip 업로드 실패 | **높음** | 방법을 모름 + 복구 경로 불명확 |

### Day 1~3

| 상황 | 이탈 확률 | 원인 |
|------|-----------|------|
| 트윈 말투가 어색함 | **높음** | 첫 번째 AI 응답이 기대 이하이면 신뢰 붕괴 |

## 4. 감성 피크 순간들

**이 순간들이 제대로 연출되면 앱이 살아난다.**

### 피크 1 — 제네시스 완료 & 오라 리빌
클레이 아바타가 흐릿한 상태(Stage 0)에서 시작해 인터뷰가 진행되면서 점점 또렷해지다가(`confidence >= 0.85` → Stage 3) 최종 오라 색과 에니어그램 유형이 확정되는 순간. **이게 앱 전체에서 가장 강력한 감정적 순간**. 애니메이션이 제대로 연출된다면 소름 돋는 경험이 될 수 있음. 이 순간을 캡처해서 파트너에게 보내고 싶어지도록 설계하는 것이 바이럴 핵심.

### 피크 2 — 트윈 AI가 처음으로 "내 말투"로 대답하는 순간
`toneVector` 기반으로 이모지 빈도, 웃음체, 반말 리듬을 흉내 낸 첫 번째 트윈 응답. "와 진짜 나같다"라는 반응이 나오면 파트너에게 즉시 공유하게 됨. 이 순간이 앱의 핵심 입소문 포인트.

### 피크 3 — 커플 Wrapped 처음 열어보는 순간
`WrappedModal`에서 우리 연애의 타임라인, 대화 패턴, 오라 변화가 요약돼서 나오는 경험. 스포티파이 Wrapped처럼 "우리 Wrapped 봤어?"가 커플끼리 연례 이벤트가 될 수 있음.

### 피크 4 — Crisis Intelligence가 실제로 울릴 때
`useCrisisIntelligence` 훅이 `rapidSwing`을 감지해서 `CrisisOverlay`가 올라오는 순간. "최근 대화에서 급격한 변화가 감지됐어요. 잠깐 멈추고 서로의 감정을 확인해보세요." 이 알림을 받고 실제로 파트너와 대화를 나눴다면, 그 커플은 영원한 팬이 됨.

## 5. 경쟁 앱 대비 차별점 분석

| 기능/컨셉 | Twin.me | 커플 다이어리류 | 관계 상담앱류 |
|-----------|---------|----------------|--------------|
| 에니어그램 기반 성격 추론 | ✅ 베이지안 실시간 | ❌ | 일부 설문형 |
| 오라 색 = 나의 정체성 | ✅ 독자적 시스템 | ❌ | ❌ |
| 내 말투 복제 AI | ✅ toneVector | ❌ | ❌ |
| 카카오톡 대화 분석 | ✅ (프라이버시 보호) | 일부 | ❌ |
| 위기 감지 알림 | ✅ | ❌ | ❌ |
| 커플 Wrapped | ✅ | 일부 포토북 | ❌ |
| 데이트코스 피드 | ✅ | ❌ | ❌ |

Twin.me가 단독으로 가진 것: **오라 시스템 + 트윈 AI 말투 복제 + 위기 감지**. 이 세 가지가 경쟁 우위의 전부임. 온보딩이 이 세 가지에 최대한 빨리 닿도록 설계되어야 함.

## 6. 구체적 개선 제안 (우선순위 순)

### P0 — 즉시 수정 필요

**① 카카오 업로드 앞에 온보딩 가이드 화면 삽입**
```
화면 구성:
- 타이틀: "카카오 대화를 가져오면 트윈이 더 나다워져요"
- 카카오 내보내기 3단계 스크린샷 or GIF
- "직접 해볼게요" / "지금은 건너뛸게요 (말투 정확도 낮아짐)" 버튼
```

**② 홈 화면 정보 계층 단순화**
```
Before: 11개 컴포넌트 동시 노출
After:
  - Fold 1 (첫 화면): 오라 클레이 아바타 + 오늘의 상태 한 줄 + 트윈 대화 CTA
  - Fold 2 (스크롤): 점수 게이지 + 무드 태그
  - Fold 3 (스크롤): AI 코칭 카드 + 메모리 링 + 공유
```

**③ 제네시스 완료 직후 파트너 초대 CTA 삽입**
```
신규 화면: "오라 리빌" 전환 애니메이션
  → "내 오라 색 공유하기" (이미지 공유)
  → "파트너도 Twin.me 해보게 초대하기" (딥링크)
  → "탭으로 이동" (스킵)
```

### P1 — 1주일 내 개선 권장

**④ 무료 플랜 채팅 제한 구조 변경**
- 설치 후 14일간 무제한
- 이후 월 10회 (현재 4회에서 상향)
- 남은 횟수를 예쁜 인디케이터로 시각화 (불안이 아니라 아끼고 싶은 감정으로)

**⑤ 연인방 잠금 상태 개선**
```
현재: 🔒 "연인을 초대해야 이용할 수 있어요"
개선:
  - 미리보기 모드: 트윈이 파트너 역할로 대화 시뮬레이션
  - "파트너 초대하면 이렇게 바뀌어요" 애니메이션 프리뷰
  - 초대 코드 생성을 연인방 화면에서 바로 할 수 있도록
```

**⑥ 연애 시작일 입력 → DatePicker 교체**
- `profile.tsx`와 `AddPlaceModal` 동일하게 적용

### P2 — 다음 스프린트

**⑦ 트윈 말투 품질 지표 노출**
- 카카오 데이터 없이 진입한 유저에게 "트윈이 나를 알아가는 중이에요 — 대화할수록 정확해져요" 진행 바 표시

**⑧ Helix 카드 실제 데이터 연동 전 플레이스홀더 처리**
- 샘플 카드에 "카카오 대화 업로드 후 실제 추억으로 채워져요" 오버레이 또는 안내 카피

**⑨ 점수 변동 스토리텔링**
- 점수가 전날보다 오르거나 내려갔을 때 원인을 한 줄로 설명하는 카드
- 예: "어제 트윈과 대화를 오래 나눠서 2점 올랐어요"

## 7. 총평

Twin.me는 **개념이 압도적으로 좋은 앱**이다. 오라 시스템, 4막 에니어그램 인터뷰, 트윈 AI 말투 복제, 위기 감지 — 이 네 가지 중 하나라도 경쟁 앱에 있으면 언급할 것인데, 전부 없음. 차별점이 명확하고 2030 커플이 원하는 "진지하지만 힙한 자기성찰"이라는 결에 정확하게 맞음.

문제는 **이 감성에 닿기까지의 여정이 너무 험난하다**는 것. 카카오 zip 장벽, 정보 과밀 홈, 파트너 미연동 공백, 조기 유료화 벽 — 이 네 가지가 제거되지 않으면 "좋은 앱인데 아무도 쓰지 않는 앱"이 될 수 있음.

온보딩 완주율이 70%를 넘는 순간, 그리고 트윈 AI 말투 복제 품질이 "와 진짜 나같다"를 안정적으로 만들어내는 순간 — 이 두 가지가 리텐션을 폭발시키는 트리거가 될 것이다.

**이 앱, 제대로 완성되면 진짜 유명해질 수 있음.**

```
마지막 한 마디 (하준 & 서연):
"다음 업데이트 때 파트너 연동 제대로 되면 우리 같이 쓸 것 같아."
```
