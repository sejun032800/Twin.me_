## 1. 무엇이 좋아졌나

### ✅ 웰컴 화면 — DNA 글로우 링, 완전히 달라진 첫인상

**v1과 가장 크게 달라진 화면.** 다크 배경(#0A0D1A)에 민트 글로우 링과 코랄 내부 원 위에 🧬 이모지를 얹은 구조가 앱 정체성을 첫 화면에서 전달함. `Twin.me` 레이블을 작게 올리고 "나를 닮은 AI가 / **연애를 돌아봐요**" 타이틀이 코랄로 강조되는 방식은 카피와 비주얼이 처음으로 일치하는 느낌.

v1에서 "시작하기" 버튼 하나뿐이었던 구조가 **시작하기(코랄 solid) + 로그인(코랄 ghost)** 2-버튼 계층으로 분리된 것도 명확해짐. 게스트 버튼은 최하단 작은 텍스트로 내려가서 의도적 선택이 되도록 처리한 것도 올바른 방향.

```
느낀 점 (서연): 진짜 앱 느낌난다.
예전엔 텍스트만 있어서 "이게 뭐야?" 했는데
이번엔 이 화면만으로도 뭔가 있겠다 싶었음.
```

**남아 있는 문제:** 웰컴 화면의 subtitle — "카카오톡 대화로 트윈 AI를 만들고 나의 소통 패턴을 함께 성찰해요"는 기능 설명은 맞는데 감성이 없음. "성찰"이라는 단어는 2030 대화에서 잘 안 쓰는 어휘. "나를 더 잘 알아가는 연애"처럼 더 구어체로 바꾸는 게 좋겠음.

### ✅ 카카오 가이드 화면(`kakao-guide.tsx`) — P0 이탈 포인트 해소

v1 리뷰에서 "온보딩 최대 이탈 포인트"로 지목한 문제를 정면으로 해결함. 4단계 스텝 카드 구조:

| 스텝 | 내용 |
|------|------|
| 1 | 💬 카카오톡 앱 열기 — "어떤 채팅방이든 괜찮아요" |
| 2 | ⚙️ 우측 상단 메뉴 탭 — "설정 → 대화 내용 → 대화 내보내기" |
| 3 | 📁 "파일로 저장" 선택 — "zip이면 더 좋지만 txt도 가능해요" |
| 4 | 📤 파일을 Twin.me로 가져오기 |

상단 프로그레스 바(코랄 fill)와 스텝 도트(완료 = 민트, 현재 = 코랄 pill 형태)로 진행감을 시각화한 것도 좋음. "카카오 없이 시작할게요" 스킵 버튼이 가장 하단 작은 텍스트로 존재하는 구조는 의도적 다운플레이가 잘 됨.

```
느낀 점 (하준): 이거 있었으면 처음에 안 막혔을 것 같은데.
경로가 정확히 나와 있어서 서연도 보고 바로 따라할 수 있겠다.
```

**남아 있는 문제들:**

첫째, **이미지/GIF가 없음.** 텍스트 설명만으로는 "우측 상단 ≡ 아이콘"이 어디 있는지 비개발자 유저에게 여전히 모호함. 각 스텝 emojiCard 자리에 실제 카카오톡 스크린샷 스타일 일러스트(크롭된 이미지)가 있으면 완성도가 확 올라감.

둘째, **스텝 2의 경로 표기가 실제 카카오와 다를 수 있음.** "설정 → 대화 내용 → 대화 내보내기"는 iOS 기준이고 Android에서는 메뉴 구조가 다름. 두 OS 기준을 명시하거나 가장 보편적인 경로만 남기는 게 안전함.

셋째, **스킵 후 트레이드오프 설명 없음.** "카카오 없이 시작할게요" 탭 시 genesis로 바로 이동하는데, 이때 트윈 말투 정확도가 낮아진다는 안내가 없음. v1 리뷰에서 지적한 그대로 미수정.

### ✅ 제네시스 완료 → `invite-hook.tsx` — 가장 큰 개선, 바이럴 루프 생성

v1 리뷰에서 "가장 큰 바이럴 미스"로 지목했던 포인트를 정확히 해결함.

`genesis.tsx`의 `handleStart()` 완료 후 `router.replace('/(tabs)')` 대신 `router.replace('/(auth)/invite-hook')`으로 라우팅 변경. `invite-hook.tsx`는:

- 오라 dominant 색으로 배경 글로우를 동적 생성 (`auraTint()` 함수)
- 🪞 이모지 중심에 "**{name}의 트윈이 완성됐어요**"
- 실제 공유 카드 미리보기 (인라인 프리뷰 카드, 오라 색 배지 포함)
- "💌 연인에게 보내기" → `Share.share()` 시스템 공유 시트
- "나중에 할게요 — 앱 먼저 둘러볼게요" 스킵 버튼
- **"연인이 없어도 혼자서 모든 기능을 쓸 수 있어요"** 안내 힌트 텍스트 ← 이거 진짜 중요

오라 색이 프리뷰 카드 border, 아바타 배경, 배지 텍스트에 일관되게 쓰이는 게 시각적 통일감을 만들고 공유 욕구를 높임.

```
느낀 점 (서연): 와 이거 예쁘다. "내 오라 색 보기 →" 배지가 내 색이랑
맞아서 신기했음. 하준한테 바로 공유함.
하준 반응: "나도 해야겠다고 했음"
→ 파트너 초대 루프 실제로 발동
```

**남아 있는 문제들:**

첫째, `Share.share()`의 메시지 텍스트가 하드코딩 — "우리 연애 DNA 일치율 확인해볼래? Twin.me 다운받고 초대 코드 입력해줘 💌". 그런데 이 메시지에 **실제 초대 코드가 없음.** 상대방이 메시지를 받고 앱을 깔아도 코드를 어디서 찾아야 할지 모름. `inviteCode`를 `coupleStore`에서 가져와 메시지에 포함시켜야 함.

둘째, "내 오라 색 보기 →" 배지를 누를 수 있을 것 같은데 `TouchableOpacity`가 아닌 일반 `View`. 탭 했을 때 아무 반응 없으면 어색함.

셋째, 이 화면에 **뒤로가기 제스처를 막는 처리가 없음.** 유저가 뒤로 스와이프하면 제네시스 화면으로 돌아갈 수 있는데 이미 완료된 인터뷰로 돌아가는 건 혼란스러움.

### ✅ 홈 탭 — Above Fold / Below Fold 구조 분리

v1의 핵심 문제였던 "11개 컴포넌트 동시 노출"이 개선됨. 새 구조:

**Above Fold (첫 화면):**
1. 헤더 — 이름 + "연애 N일째 · 날씨" (한 줄로 합침)
2. 히어로 — ClayTwinAvatar (80px → 120px로 확대) + 오라 glow
3. 점수 게이지 + 티어
4. 무드 태그 3개
5. "아래로 스크롤해보세요 ↓" 구분 힌트

**Below Fold (스크롤 후):**
6. AI 코칭 카드 (코랄 tint 래퍼로 감싸서 구분)
7. 메모리 링
8. 히스토리 / 공유하기 액션 버튼
9. 파트너 상태 바
10. 오버플로우 배너 (조건부)

**하단 고정 CTA:** "트윈과 대화하기 →" 코랄 버튼 (ScrollView 밖)

v1의 `ShareButton`이 홈 중앙에 있던 것이 액션 버튼 행으로 이동하고, 하단 CTA가 항상 보이는 구조가 됨. 게이지 크기가 220px 유지에 아바타가 120px로 커지면서 히어로 섹션의 무게감이 생긴 것도 좋음.

```
느낀 점 (하준): 이제 첫 화면에서 뭘 눌러야 할지 명확해졌음.
밑에 고정 버튼이 있어서 자연스럽게 트윈방으로 이동함.
```

**남아 있는 문제들:**

첫째, **"아래로 스크롤해보세요 ↓" 힌트가 너무 직접적이고 색이 너무 연함.** `#CCCCCC` 텍스트는 라이트 모드에서 간신히 보이고 다크 모드에서도 존재감이 없음. 이 힌트를 텍스트로 명시하는 것 자체가 어색함. Lottie 화살표 애니메이션으로 교체하거나, 첫 실행 때만 보이도록 조건부 처리하는 게 나음.

둘째, **홈 화면 배경색이 `#FBF8F3` (크림)이고 하단 CTA 배경도 `#FBF8F3`인데 다크 모드에서 이게 `theme.bg`를 쓰지 않고 하드코딩되어 있음.** `bottomCTA`와 `actionBtn`의 `backgroundColor: '#FFFFFF'`는 다크 모드에서 흰색 섬이 생기는 버그. `theme.card` / `theme.bg` 토큰으로 교체 필요.

셋째, **점수가 왜 지금 이 숫자인지 여전히 설명이 없음.** v1 리뷰에서 "점수 변동 스토리텔링"을 P2로 제안했는데 미구현. 게이지 아래에 "트윈과 3번 대화해서 2.4점 올랐어요" 같은 한 줄 설명이 있으면 숫자가 살아남.

넷째, **PartnerStatusBar가 Below Fold로 내려갔는데** 파트너 미연동 유저에게는 초대 유도 배너가 스크롤 아래에 묻혀버림. 파트너 연동은 앱의 핵심 행동인데 위계가 낮아짐. 연동 여부에 따라 위치를 다르게 하는 조건부 처리가 필요함 (미연동 → Above Fold 상단, 연동 → Below Fold).

## 2. 변경되지 않은 것들 — 남은 과제

### ❌ 채팅 탭 — `FREE_MONTHLY_LIMIT = 4` 그대로

v1 리뷰에서 "진심 생기기 전에 먼저 벽이 생기는 구조"로 지목한 문제가 그대로임. 코드 38번째 줄: `const FREE_MONTHLY_LIMIT = 4;` 변경 없음. Alert 카피도 "무료 플랜은 이번 달 4회 대화가 가능해요."로 동일.

설치 후 2주 내에 AI 응답의 말투 품질을 확인하고, Muse Sheet를 써보고, 분석가방에서 리포트를 한 번 받아보는 데 4회는 부족함. 특히 첫 1-2회는 테스트 대화로 소비되는 경우가 많음.

```
느낀 점 (하준): 트윈방 두 번 써보고 세 번째에 막힘.
아직 말투가 정확한지도 모르겠는데 이미 한도 다 씀.
업그레이드 할 이유를 못 느낀 상태에서 벽이 생기니까 그냥 닫음.
```

**여전히 필요한 것:**
- 설치 후 14일은 무제한 또는 월 10회 이상으로 상향
- 남은 횟수를 "이번 달 N회 남았어요" 형태로 입력창 위에 작게 상시 노출
- 한도 소진 Alert 카피를 "아직 4번밖에 안 써봤는데 업그레이드는 고민이 됨" 느낌으로 재작성


FeedTab의 "📋 샘플 데이터입니다. Supabase 테이블 생성 후 실제 코스가 표시돼요." 배너도 유지 중. 개발 메모가 앱 UI에 그대로 노출되는 것은 출시 전 반드시 정리 필요.

## 3. 새로 발견된 문제들

### 🆕 invite-hook — 초대 코드 미포함 공유 메시지

앞서 언급했지만 가장 중요한 신규 버그. `invite-hook.tsx`의 Share 메시지:

```tsx
message: `${name ?? '나'}의 트윈 AI가 완성됐어 🧬\n우리 연애 DNA 일치율 확인해볼래?\nTwin.me 다운받고 초대 코드 입력해줘 💌`
```

초대 코드가 없음. 파트너가 이 메시지를 받고 앱을 설치해도 코드를 알 수 없어서 커플 연동을 못 함. 바이럴 루프를 연결하는 마지막 한 줄이 빠진 것. `useCoupleStore`에서 `inviteCode`를 가져와 동적으로 삽입해야 함. 단, 아직 커플 코드가 없는 상태(파트너 연동 전 단계)라면 이 화면 진입 시 코드를 자동 생성해서 store에 저장하는 로직도 필요함.

```tsx
// 개선 예시
const inviteCode = useCoupleStore((s) => s.inviteCode) ?? generateInviteCode();

message: `${name}의 트윈이 완성됐어 🧬
초대 코드: ${inviteCode}
Twin.me 설치 후 코드 입력하면 우리 일치율 볼 수 있어 💌`
```

---

### 🆕 홈 화면 하드코딩 색상 — 다크 모드 버그

`makeStyles()`의 여러 지점에서 테마 토큰 대신 하드코딩 색상 사용:

```tsx
actionBtn: {
  backgroundColor: '#FFFFFF',  // ← 다크모드에서 흰색 섬
  borderColor: 'rgba(0, 0, 0, 0.06)',  // ← 다크모드에서 안 보임
},
bottomCTA: {
  backgroundColor: '#FBF8F3',  // ← 다크모드에서 크림색이 떠버림
},
```

라이트 모드에서는 예쁘지만 다크 모드(또는 6시그마 테마) 전환 시 흰색 카드와 크림 배경이 떠버리는 버그.

교체 방법:
- `'#FFFFFF'` → `theme.card`
- `'#FBF8F3'` → `theme.bg`
- `'rgba(0, 0, 0, 0.06)'` → `theme.border`

### 🆕 kakao-guide → kakao-upload 라우팅 체인이 길어짐

v2 온보딩 플로우: `welcome → signup → profile → kakao-guide → kakao-upload → genesis → invite-hook → tabs`

총 8단계. 카카오 가이드 추가로 진입 장벽은 낮아졌지만 단계가 늘어남. 특히 `kakao-guide`(4스텝 내부 페이지네이션) + `kakao-upload`가 연속으로 오면서 총 소요 시간이 길어짐. 카카오를 건너뛰는 유저도 `kakao-guide`를 통과해야 하는 구조가 약간 비효율적. `kakao-guide`의 스킵 버튼이 genesis로 바로 점프하는 건 맞지만, 이 화면 자체를 건너뛸 수 있는 옵션이 없음.

**제안:** profile 화면에서 "카카오 대화 가져오기" / "건너뛰고 바로 시작하기" 선택지를 먼저 보여주고, 선택에 따라 `kakao-guide → kakao-upload` 또는 `genesis`로 분기하는 구조.

## 7. 총평

v2는 **올바른 곳을 고쳤다.** 카카오 가이드, 제네시스 완료 후 invite-hook, 홈 Above/Below Fold 구조 — 이 세 가지는 v1 리뷰에서 가장 중요하다고 짚은 포인트였고 전부 구현됐음. 특히 `invite-hook.tsx`는 오라 색을 동적으로 적용한 글로우 UI와 공유 카드 미리보기까지 넣은 게 인상적이고, 이것만으로도 앱 바이럴 잠재력이 실제로 올라감.

남은 숙제는 크게 두 가지. 하나는 **invite-hook의 공유 메시지에 초대 코드가 없다는 것** — 바이럴 루프의 마지막 고리가 빠진 것이라 빠르게 수정해야 함. 다른 하나는 **무료 채팅 4회 제한과 연인방 싱글플레이어 공백** — 이 두 가지는 파트너 없이 혼자 쓰는 첫 1~2주를 어떻게 채울 것인가의 문제이고, 아직 답이 없음.

```
하준의 최종 한 마디:
"이번엔 서연한테 공유까지 했음. 다음엔 진짜 같이 써볼 것 같아."

서연의 최종 한 마디:
"근데 코드가 없어서 서연이 앱 깔고 연동을 못 했음.
 코드 어디서 찾아요? 👀"
```
