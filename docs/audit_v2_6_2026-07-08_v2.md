| 상태 | 개수 | 비율 |
|---|---|---|
| ✅ 구현 완료 | 43 | 71.7% |
| ⚠️ 부분 구현 | 16 | 26.7% |
| ❌ 미구현 | 1 | 1.7% |

**1차 감사(56개 항목) 대비 변화:** ✅ 24→43(+19) / ⚠️ 21→13(−8, 신규 3건 포함 시 순감소는 표에 반영됨) / ❌ 11→0(−11). 신규 발견 4개 항목(❌ 1, ⚠️ 3)을 합산한 60개 항목 기준 수치는 위 표와 같다.

**남아있는 공백 (우선순위순):**
1. **FUN-HOM-002B 오버플로우 배너** — 유일한 완전 미구현 항목. `CRITICAL_LOSS`/`EXCESS_GAIN` UI 트리거가 없어 CrisisMode(§4, ✅) 진입 동선 중 하나가 비어있음.
2. **Founding VIP 13개월차 자동 전환** — 코드는 있으나 호출부가 없는 유일하게 남은 "정의는 있으나 호출 안 되는" 패턴.
3. **거울방 월 사용 한도** — 스펙과 다른 근사치(일 단위 4회)로 구현되어 있어 정책 정합성 재검토 필요.
4. **색상 토큰/그라데이션/`.toFixed(1)` 잔존 위반** — 규모는 크게 줄었으나(57→31건, 위반 지점 7곳) 완전히 해소되지는 않았고, 신규 코드(Wrapped/리포트)에서 동일 패턴이 재발하는 추세가 확인됨 — 신규 컴포넌트 작성 시 체크리스트화가 필요해 보임.
5. **IAP/react-native-iap Expo Go 비호환** — 1차 감사 이후 조치 없음, 네이티브 빌드 전환 시점 결정 필요.
6. **커플 연동 `couples` 테이블 SQL 미실행 여부** — 코드로 검증 불가능한 외부 상태로 남아있음(`date_courses` 테이블도 동일 패턴 추가됨, 다만 이쪽은 MOCK 폴백이 있어 체감 리스크는 낮음).

**가장 뚜렷한 개선 패턴:** 1차 감사가 반복적으로 지적한 "정의는 있으나 호출되지 않는 죽은 코드"(coolingBleed, computeVolatilityIndex, detectRapidSwing, usePremiumGate, CrisisMode, PartnerStatusBar, weeklyReportService, ShareCard/WrappedModal 관련 전체)가 이번 점검 시점에는 `reconcileFoundingVipExpiry` 단 하나를 제외하고 전부 실제 호출부를 갖게 됨. 엔진/서비스 레이어와 UI를 잇는 "마지막 연결 단계"가 이번 작업분에서 집중적으로 메워진 것으로 보인다.
