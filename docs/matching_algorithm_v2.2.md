## 2. 실시간 틱 엔진 (Real-Time Tick Engine)

### 2.1 점수 2층 구조

- **`S_Live`** — 화면 상단 게이지. 이벤트가 감지될 때마다 즉시 재계산되어 애니메이션으로 출렁임. 하루에 수십 번 변동. **표시 전용(non-persistent)**.
- **`S_Current`** — 공식 일치율. 매일 자정 1회 정산되어 영구 저장. 10단계 티어·페이월 등 모든 시스템이 참조하는 값.
- **`S_Today_Open`** — 당일 00:00 시점의 `S_Current`. 그날 `S_Live`의 기준선.

```
S_Live(t) = S_Today_Open + Δ_intraday( A_t )
```
`A_t` = 당일 누적 이벤트 가중합(아래 2.2~2.4).

### 2.2 이벤트 단위(틱) 처리 파이프라인

메시지 스트림에서 이벤트 k가 감지될 때마다 즉시 다음을 수행한다.

📐 **[수식 2-1] 유효 변동치 (v2.1 승수 계승)**
```
δ_eff,k = δ_base,k × M_intensity × M_context × M_reciprocity × M_time
```
- `M_intensity` 0.5~1.5 (NLP 강도·확신도)
- `M_context` 1.0 / 갈등 중 1.5
- `M_reciprocity` 1.0 / 상호 1.2
- `M_time` 가산 1.2 / 감산 1.3 (00:00~04:00 취약시간)

📐 **[수식 2-2] 부정성 가중**
```
δ_w,k = (δ_eff,k > 0) ? δ_eff,k : (κ · δ_eff,k)     // κ = 1.5
```

📐 **[수식 2-3] 반(反)파밍 — 동일 코드 체감 감쇠**

같은 코드가 당일 n번째로 발생하면 가중이 기하급수적으로 줄어든다(스팸 방지).
```
δ_final,k = δ_w,k · γ^(n−1)          // γ = 0.5, n = 당일 해당 코드 발생 순번
단, 코드별 일일 빈도 캡 N 초과 시 δ_final,k = 0
```

> 예: `G-HUM-001 드립 성공` → 1회 +0.10 / 2회 +0.05 / 3회 +0.025 … (N=5 초과분 0)
> **중대 이벤트(L-CRU·L-HRS)는 단일 발생으로도 큰 폭 변동을 일으키도록 의도적으로 별도 감쇠를 적용하지 않거나 γ를 0.8로 완만하게 둔다.**

📊 **[수식 2-4] 누적 어그리게이터 (실시간 갱신)**
```
A_t = A_(t-1) + δ_final,k        // 이벤트마다 즉시 갱신 → S_Live 재렌더
```

### 2.3 tanh 소프트 포화형 일일 제한 (하드 클램프 폐기)

딱딱한 벽 대신 **점근(asymptote) 곡선**으로 일일 총량을 부드럽게 가둔다. 초반 이벤트는 점수를 잘 움직이고(거의 선형), 쌓일수록 한 틱당 마진이 줄어든다.

📐 **[수식 2-5] 일일 등락 변환 함수**
```
Δ_intraday(A) = cap · tanh( A / cap )

  cap = (A ≥ 0) ? A_cap⁺ : A_cap⁻
  A_cap⁺ = 2.0     (일일 가산 점근 상한)
  A_cap⁻ = 2.5     (일일 감산 점근 하한, 비대칭)
```

특성:
- `|A|`가 작을 때 `Δ ≈ A` → **반응성 100%**(수십 번 출렁이는 구간).
- `|A|`가 커질수록 `Δ → ±cap` → **부드러운 포화**(농사·폭주 방지).
- κ가 감산을 미리 키우고 `A_cap⁻ > A_cap⁺`까지 더해져 **'손상은 빠르다' 원칙**을 이중으로 보장.

> 제한 폭 재설정 요약: 기존 `±1.0 / −1.5` 하드 → **`±2.0 / −2.5` 소프트(tanh)**. 실효 하루 변동 폭은 넓어졌지만, 벽에 '쾅' 부딪히는 느낌 없이 자연 수렴.

### 2.4 자정 정산 & 항상성 감쇠

📐 **[수식 2-6] 일일 정산**
```
Δ_Daily   = Δ_intraday(A_endOfDay)
Δ_Decay   = −λ · ( S_Today_Open − S_Master_Base )       // λ = 0.02
S_Current = Clamp( 50.5 , 100.0 , S_Today_Open + Δ_Daily + Δ_Decay )

→ 다음날 S_Today_Open = 갱신된 S_Current, A_t 초기화(0)
```

- **글로벌 하한 50.5 / 인터뷰 전 상한 89.5 / 글로벌 상한 100.0** (v2.1 가드 계승)
- 항상성 감쇠 `λ`로 점수가 성격 기준점으로 천천히 회귀 → 단발 이벤트는 자연 희석, 지속 노력은 누적 보상.

### 2.5 (옵션) 실시간 냉각 블리드

이벤트 없이 일정 시간(예: 90분) 정적이면 `A_t`가 0 방향으로 미세 감쇠하여 `S_Live`가 중립으로 식는 연출(`A ← A · e^(−β·Δt)`, β 소). **표시 전용 연출**이며 정산값에는 `A_endOfDay`만 반영. 도입 시 "대화가 식으면 게이지도 식는다"는 직관 제공(과한 패널티로 느껴지면 β를 0으로 비활성).

## 3. 오버플로우 & 실시간 위기 감지

### 3.1 상태 분기 (심리 임계 ±1.0 고정)

포화 캡(±2.0/−2.5)과 별개로, 배너 트리거 기준선은 **±1.0%**로 유지한다(FUN-HOM-002 일관성).

🚨 **[수식 3-1]**
```
Overflow_Status =
  EXCESS_GAIN     if  A_endOfDay >  +1.0
  CRITICAL_LOSS   if  A_endOfDay <  −1.0
  NONE            otherwise

Spillover = A_endOfDay − Δ_Daily        // tanh가 흡수(절단)한 잉여분
```

### 3.2 심각도 3단계 (`|A_endOfDay|` 기준)

| 심각도 | 조건 | 연출 |
|---|---|---|
| MINOR | 1.0 ~ 1.75 | 옅은 펄스 배너 |
| MAJOR | 1.75 ~ 2.5 | 그라데이션 보더 배너 |
| CRITICAL | 2.5 초과 (포화 절단 발생) | 전면 글래스모피즘 + 햅틱 |

### 3.3 실시간 급락 감지 (Rapid-Swing Trigger) — 신규

자정 정산을 기다리지 않고, **단시간 내 급격한 `A_t` 하락**을 즉시 포착한다.

📐 **[수식 3-2]**
```
if ( A_(t) − A_(t−Δw) ) < −1.5   within   Δw ≤ 30분
    → 즉시 FUN-CHA-003(반성의 거울) 민감도 상승 / 조건 충족 시 강제 발동
```

### 3.4 변동성 지수 & 누적 위기 메모리

- `V = StdDev(틱 간 δ_final over 최근 24h)` → 기복 클수록 크라이시스 트리거 민감도 동적 상향.
- `CRITICAL_LOSS` 3일 연속 → 위기 메모리 ON → 임시로 `A_cap⁺`를 1.0으로 하향(과잉 보정 방지).
- `EXCESS_GAIN` Spillover → 점수 미반영, 대신 **'예쁜 말' 아카이브 크레딧** 적립 → 추억 월 라우팅.

## 5. 통합 연산 예시 (Tick 기반)

```
S_Master_Base = 81.6,  S_Today_Open = 81.6,  A = 0
A_cap⁺ = 2.0, A_cap⁻ = 2.5, κ = 1.5, λ = 0.02
```

**오전~저녁 (다정함 누적)**
```
G-RES-002 +0.05, G-HUM-001 +0.10, G-REG-009 +0.08, G-INT-002 +0.08, ... (다수)
A ≈ +0.90
S_Live = 81.6 + 2.0·tanh(0.90/2.0) = 81.6 + 0.844 = 82.4   ← 게이지 상승
```

**21:00 갈등 발생**
```
L-CRU-002 −0.30 × κ1.5 = −0.45 → A = 0.45
S_Live = 81.6 + 2.0·tanh(0.225) = 81.6 + 0.444 = 82.0      ← 게이지 급락(−0.4)
  └ Rapid-Swing: 30분 내 −0.45 (임계 −1.5 미만) → 경보 대기
```

**21:30 화해 + 콤보**
```
G-CON-001 +0.25 × M_ctx1.5 = +0.375 → A = 0.825
C-ARC-001 콤보 +0.20 → A = 1.025
S_Live = 81.6 + 2.0·tanh(0.5125) = 81.6 + 0.943 = 82.5      ← 회복
```

**자정 정산**
```
Δ_Daily = 2.0·tanh(1.025/2.0) = +0.943
Δ_Decay = −0.02·(81.6 − 81.6) = 0
S_Current = Clamp(50.5, 100.0, 81.6 + 0.943 + 0) = 82.5
Overflow: A(1.025) > +1.0 → EXCESS_GAIN / MINOR
  └ Spillover = 1.025 − 0.943 = 0.082 → 아카이브 크레딧 적립
다음날 S_Today_Open = 82.5, A = 0
```

## 6. 튜닝 파라미터 요약

| 파라미터 | 기본값 | 역할 |
|---|---|---|
| `A_cap⁺ / A_cap⁻` | +2.0 / −2.5 | tanh 일일 포화 캡(비대칭) |
| `κ` | 1.5 | 부정성 가중 |
| `γ` | 0.5 (중대 0.8) | 동일 코드 체감 감쇠 |
| 코드별 빈도 캡 `N` | 코드별 상이 | 반파밍 하드 백스톱 |
| `M_intensity` | 0.5~1.5 | 신호 강도 |
| `M_context` | 1.5 (갈등 중) | 상황 가중 |
| `λ` | 0.02 | 항상성 감쇠 |
| Rapid-Swing | −1.5 / 30분 | 실시간 위기 감지 |
| 글로벌 가드 | 50.5 ~ 100.0 | 점수 절대 범위 |

**구현·윤리 주의**
- 모든 수치는 재미·자기성찰 도구로 포지셔닝(과학적 진단 오인 방지 카피 필수).
- L-HRS·중대 지표의 텍스트 분류는 **근사**이며 오탐 시 관계에 해가 될 수 있으므로 보수적 임계 설정.
- `δ`·`κ`·`γ`·`A_cap`은 출시 후 실로그 A/B 보정 대상(초기값은 출발점).
- 온디바이스 선처리(민감정보 마스킹·상대 발화 Drop)는 모든 입력 단계 선행.
