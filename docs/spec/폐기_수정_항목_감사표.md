## 요약

총 19개 항목을 조사했다. **폐기 7건(37%), 수정 12건(63%)** — 완전 신규 제거·대체가 필요한 항목보다 구조는 두고 내부 로직·스키마만 교체하면 되는 항목이 더 많지만, 폐기 7건 중 4건(`genesisInference.ts` 전체, `TYPE_AXIS_LEANING` 상수, `BayesianState` 타입, `useGenesisInterview.ts` 상태머신)이 서로 강하게 결합되어 있어 사실상 인터뷰 파이프라인의 심장부를 통째로 들어내는 작업이다. 가장 리스크가 큰 변경은 두 가지다. **(1) `genesisInference.ts`는 애니어그램 코어(Δ⁹) 단일 축만 다루는 폐쇄형 베이지안 엔진**이라 Big5·애착·날개·스턴버그까지 다루는 v2.1 다차원 사후분포로 교체하면 이를 소비하는 `useGenesisInterview.ts`, `genesisQuestionBank.ts`, `userStore`, `genesis.tsx` 화면까지 연쇄 수정이 불가피하다. **(2) 인터뷰에 Gemini 연동이 아예 없다** — 구현명세서는 기존 파이프라인이 "자연어 답변 → 구조화 값" 변환을 이미 어떤 형태로든 하고 있다고 가정했지만, 실제로는 `matchArchetype()`이라는 순수 키워드 부분일치 매칭만 있고 LLM 호출은 전무하다. 이는 "대체"가 아니라 "신규 구축"에 가까운 작업량을 의미한다.

표 밖 참고 — 애매하지만 "유지"로 판단한 항목: `getTierFromScore`/`TIER_THEMES`(10단계 티어 UI)는 점수 범위(50~100)가 v2.1 `DNA%`와 호환되고 v2.1 문서가 UI 계층을 규정하지 않으므로 그대로 두되, 입력값만 새 `DNA%`로 교체하면 될 것으로 보인다. `scoreStore`의 `sLive`/`dailyStatusHistory`/`eventLog`/`crisisMemoryActive`/`volatilityIndex`(실시간 틱 엔진, 자정 정산, 냉각 소실)는 v2.1이 전혀 다루지 않는 별개 시스템(`docs/Twin_me_MASTER_v2.6.md §5`)이라 이번 개정과 무관하다. `couples` 테이블은 `CREATE TABLE` 정의 자체가 코드 어디에도 없는(콘솔 수동 생성 추정) 기존 리스크이지만 이번 v2.1 개정이 새로 만든 문제는 아니다.
