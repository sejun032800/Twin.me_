# 📑 FUN-HOM-001 (Override) — 트윈 제네시스 인터뷰 (음성 성향 인터뷰)

> **문서 성격:** 기존 `Twin.Me.md`의 FUN-HOM-001(10분 인터뷰)을 **완전 대체(OVERRIDE)** 한다.
> **핵심 전환:** '검사'가 아니라 **'내 일부가 떨어져 나와 트윈으로 태어나는 체험'**. 인터뷰어는 트윈 본인.
> **산출물:** ① `User_Persona_Matrix`(에니어/MBTI/애착/방어기제 + 6색) ② `User_Tone_Vector` 보강(발화 학습) ③ 완성된 '나의 트윈'.
> **연동:** `Chat_logic.md`(말투) · `Twin_response_logic.md`(응답) · v2.2 엔진(점수) · FUN-CHA-001(트윈 정의).
> **불변 원칙:** 모든 출력은 **단정이 아닌 제안+확인**, **따뜻한 카피**(상처 주는 라벨 금지), 온디바이스 선처리.

## 0. 설계 철학 (확정 요약)

- **프레이밍:** 성경 창세기 메타포 — 사용자의 일부를 떼어 트윈(점토)을 만든다. 트윈은 "갓 태어나 자기 근원(나)을 더듬어 알아가는" 존재.
- **트윈 = 거울:** "나는 곧 너야"를 천진하게 반복. 나중의 꾸짖음을 "또 다른 내가 말하는 것"으로 수용하게 하는 복선.
- **체험의 보상:** 내가 모르던 나(특히 에니어그램)를 트윈이 대화로 찾아줌.
- **부담→체험 전환:** 평가가 아니라 대화 / 채점이 아니라 공감 / 끝을 모르는 게 아니라 진행이 보임 / 노동의 결과물(트윈)이 손에 잡힘.

## 1. 입력·포맷 사양

### 1.1 음성 우선 + 타이핑 안전망

- **기본은 음성(STT).** "통화" 느낌의 몰입.
- **타이핑 선택권:** 화면 하단에 작고 은은한 키보드 아이콘 상시 노출(주인공은 음성).
- **무음 자동 폴백:** 음성으로 시작 후 5~10초 무발화 시, 트윈이 "말로 하기 어려우면 적어줘도 돼"라며 키보드를 부드럽게 제시.
- **혼합 허용:** 한 답은 음성, 다음 답은 타이핑 자유 전환(환경 대응).
- **STT 보정 confirm:** STT 결과를 트윈이 가볍게 되짚음("아 ~란 거지?") → 인식 오류 보정 + 공감 연출 동시.

### 1.2 사전 입력 데이터 (온보딩 FUN-ONB-001에서)

- MBTI: 자가보고(있음 / 모름).
- 에니어그램: 자가보고(있음 / **모름 = 디폴트 가정**).
- → 인터뷰 임무가 입력 상태에 따라 분기(§4.0).

## 2. 트윈 성장 시스템 (점토 메타포)

### 2.1 성장 4단계 (인터뷰 4막과 동기화)

| 단계 | 시점 | 형상 | 비고 |
|---|---|---|---|
| 0 무정형 점토 | 시작 전 | 흐릿한 회색 점토 덩어리, 미동 | "아직 아무것도 아닌 상태" |
| 1 쿠키 실루엣 | 1막(잡담) 후 | 납작한 사람 형태(진저브레드) | 초반 후킹 |
| 2 입체화 | 2~3막 진행 중 | 부풀며 3D, 부위별 또렷해짐 | 진행 바 대신 신체 완성 |
| 3 나의 트윈 | 4막 완료 | 성별/연령대 형상 + 눈 뜨고 첫 인사 | 정확도 95% 해제 |

### 2.2 "먹고 큰다" 연출

- **응답→흡수:** 발화 종료 시 말이 빛 입자/점토 조각으로 트윈에게 빨려 들어가고 트윈이 꿀렁 부풂. (내 말=양분 1:1 매칭)
- **감정 색 입히기:** 응답 감정 톤에 따라 점토가 물듦(§6 색 시스템과 동일 `w`).
- **마이크로 리액션:** 끄덕임·옴찔거림 = 경청·공감의 시각화.
- **구현 주의:** 실시간 3D 모핑은 RN 난이도 높음 → MVP는 단계별 일러스트/Lottie 전환 권장. 감동은 전환의 부드러움에서 나옴.

### 2.3 기본 성향 (‘나’를 입기 전) — 확정 3종

1. **무한한 호기심** — 존재 이유. 인터뷰어 역할과 일치. "이건 뭐야? 더 알려줘!"
2. **무조건적 애정·편들기** — "너의 일부니까 너를 사랑할 수밖에." 꾸짖음의 안전장치(미운 마음 0%인 애의 쓴소리는 걱정으로 들림).
3. **말랑한 감정 반응** — 점토답게 감정이 즉각 몸으로(좋으면 부풀고 슬프면 쪼그라듦). 투명함 → 사용자도 마음 엶.

> 이 3종은 '특정 인간형'이 아니라 '갓 태어난 보편적 상태'라, 어떤 사용자 성향과도 충돌하지 않고 '나'를 입을 여백을 남김.

### 2.4 디폴트 말투

- 중립적·따뜻한 **호기심체 반말**(표준어 기반, 딱딱하지 않게). 존댓말 X, 과한 유행어 X.
- 예: "안녕! 나는… 너의 일부래. 근데 아직 너를 잘 몰라. 알려줄래?"
- 인터뷰 진행될수록 내 말투(`User_Tone_Vector`)가 섞여 들어감(§5 블렌딩).

## 3. 베이지안 추론 엔진 (성향 판단의 심장)

### 3.1 두 개의 확률 분포

```
P_enn  = [p₁..p₉]            // 에니어 9유형 확률, Σ=1
P_mbti = {E/I, S/N, T/F, J/P} // 각 축 [0,1] (자가보고가 맞을 확률)
```

### 3.2 사전 확률 (Prior)

📐 **[수식 3-1] 에니어 '모름'일 때 MBTI 기반 사전**
```
p_i^(0) = softmax( β · affinity(MBTI_self, type_i) )
```
- `affinity`: 16×9 MBTI×에니어 상성 행렬(부록 A.1). β = 사전 뾰족함(온도).
- 에니어 '있음'이면 해당 유형에 사전 가중, 균등분포(1/9)보다 빠르게 수렴.

### 3.3 우도 (Likelihood)

```
각 질문 노드에 응답별 우도 벡터 부착:
  q.likelihood[response][type_i] ∈ [0,1]   // "이 응답이 type_i에서 나올 성향"
```

### 3.4 사후 갱신 (Posterior) — 베이즈 정리

📐 **[수식 3-2]**
```
p_i ← p_i · L(r | type_i) / Z ,   Z = Σ_j p_j · L(r | type_j)
```
- 매 응답마다 곱·정규화. 가설 스위치가 자연 발생(증거 누적 방향으로 분포 이동).

### 3.5 확신도 (Confidence) — 엔트로피 기반

📐 **[수식 3-3]**
```
H = − Σ p_i·log(p_i) ,  H_max = log(9)
Confidence = 1 − H / H_max          // 0(혼란) ~ 1(확신)
```

📐 **[수식 3-4] 보조 — 1·2위 격차**
```
Margin = p_(1st) − p_(2nd)          // 작으면 "팽팽" → 단정 금지
```

### 3.6 MBTI 오판 감지 (별도 트랙)

📐 **[수식 3-5]**
```
conflict_axis = | self_report_axis − observed_axis |   // observed = 응답 신호 가중평균
conflict ≥ θ_conflict(0.6) → 오판 의심 플래그 → 해당 축 반증 질문 추가
```

### 3.7 가설 스위치

```
직전 1위 유형 변경 시 → '가설 스위치' 이벤트 → 남은 질문 풀을 새 1위의 확증·반증으로 즉시 교체
```

## 4. 인터뷰 4막 구조

### 4.0 입력 상태별 임무 분기

| 사전 입력 | 인터뷰 임무 |
|---|---|
| MBTI ○ + 에니어 ○ | 검증·교정 모드(자가보고 vs 실제 발화 모순 체크) |
| MBTI ○ + 에니어 ✗ (최빈) | MBTI 가벼운 확인 + **자원 대부분 에니어 추론에 투입** |
| 둘 다 ✗ | 둘 다 추론(최장 버전) |

### 4.1 🎬 1막 — 잡담 (워밍업, ~90초)

- 목적: 발화 장벽 넘기(답 내용 < 목소리 내기). 트윈 점토 0→1단계.
- 가벼운 질문만: "오늘 하루 어땠어?", "요즘 사소하게 기분 좋았던 거 하나만."
- 트윈은 사용자 말 속도·톤·웃음을 캘리브레이션 후 미러링.

### 4.2 🎬 2막 — 개인 성향 파악 (에니어 추론, ~3분)

- **"나라는 사람"** 차원 — 연애 한정 X, 삶 전반의 동기·두려움.
- 토너먼트 트리: ① 본능(8·9·1)/감정(2·3·4)/사고(5·6·7) **그룹 대분류** → ② 그룹 내 3유형 정밀 변별 → ③ 후보 1~2개 확인.
- 매 응답 베이즈 업데이트(§3.4). 트윈 점토 1→2단계.
- 평균 5~7문항으로 추론(조기 종료 §4.5).

### 4.3 🎬 3막 — 연애 성향 파악 (구체, ~3분)

- 2막 가설을 **연애 맥락으로 끌어와 검증·구체화**(2막→3막 데이터 흐름).
- 질문 3종 + 동적 배분(§4.4).
- 연애 특화 축 수집: 애착(불안/회피/안정), 사랑의 언어, 서운함 트리거, 갈등 첫 반응.
- 트윈 점토 2→3단계 직전(표정·색 입혀짐).

### 4.4 3막 질문 3종 & 동적 배분

- **확증(Confirmatory):** 가설이 연애에서 어떻게 발현되는지 확인.
- **반증(Disconfirmatory):** 인접·혼동 유형 변별("유사하지만 다른 질문").
- **공통(Universal):** 유형 무관 연애 필수 축(§7.1, 항상 포함).

📐 **[수식 4-1] 확신도 기반 배분**
```
Confidence ≥0.7 & Margin ≥0.2 : 확증60 / 반증20 / 공통20
Confidence ≥0.4              : 확증40 / 반증40 / 공통20
else                          : 확증25 / 반증45 / 공통30
```

📐 **[수식 4-2] 막 내 순서**
```
공통(1~2) → 확증(2~3) → 반증(1~2, 모순 시 가설 수정) → 제안+확인 마무리
```
> 반증을 너무 일찍 던지면 혼란 → 확증으로 신뢰 쌓은 뒤 삽입.

### 4.5 종료 조건

```
조기 종료 ⇔ Confidence ≥ θ_stop(0.85) AND Margin ≥ 0.25
최대 문항 도달 시 → 현재 1위 잠정 확정 + 제안+확인 검증
```

### 4.6 🎬 4막 — 엔딩 (~60초)

- 따뜻한 마무리 질문: "트윈이 연인에게 어떤 사람으로 비쳤으면 좋겠어?"
- 완성 세리머니: 점토→트윈, 눈 뜨고 내 말투로 첫 인사("안녕, 나는 너야 ㅋㅋㅋ").
- 정확도 95% 해제 + 결과 공개(§8).

## 5. 성격 블렌딩 (디폴트 → '나') — 그라데이션 전환

### 5.1 믹싱 계수 = 확신도 재활용

📐 **[수식 5-1]**
```
w = Confidence                       // 0=완전 디폴트, 1=완전 나
트윈 성격 = (1−w)·디폴트 + w·사용자성향
```

### 5.2 3채널 차등 속도 블렌딩

📐 **[수식 5-2]**
```
w_channel = Confidence ^ γ_channel
  γ_말투 = 0.7 (빨리 물듦)
  γ_감정 = 1.0
  γ_태도 = 1.5 (천천히, 신중)
```
- **말투 채널:** 디폴트 호기심체 → 내 어미·드립(`User_Tone_Vector`).
- **감정 채널:** 디폴트 과장 → 내 정서 강도(6색 축).
- **태도 채널:** 디폴트 순수 호기심·애정 → 내 에니어/애착.

### 5.3 부드러운 추종 & 디폴트 잔향

📐 **[수식 5-3]**
```
w_display ← w_display + η·(w_target − w_display)   // η=0.1, 늘 은은하게 따라감
```
📐 **[수식 5-4] 완성 후 잔향 floor**
```
호기심 min 0.15 / 무조건적 애정 1.0(불변) / 말랑함 0까지 자유
```
> 호기심 한 스푼 → 계속 나를 관찰(거울). 애정 보존 → 쓴소리가 사랑으로 수용.

### 5.4 통합 노브

`w` 하나가 **색 차오름 + 점토 진화 + 성격 블렌딩**을 동시 구동 → "말할수록 색이 차고·점토가 자라고·얘가 나처럼 군다"를 한 흐름으로 체감.

## 6. 6색 성향 색 시스템

### 6.1 6개 성향 축 ↔ 6색

| 축 | 스펙트럼 | 색(예시, 뮤트 파스텔) |
|---|---|---|
| 애착 안정성 | 불안 ↔ 안정 | 코랄 ↔ 세이지 |
| 갈등 반응 | 직면 ↔ 회피 | 선명 ↔ 흐릿(명도) |
| 사랑의 표현 | 표현형 ↔ 내재형 | 피치 계열 |
| 정서 강도 | 잔잔 ↔ 격정 | 채도로 표현 |
| 관계 주도성 | 리드 ↔ 서포트 | 더스티 블루 |
| 신뢰 기질 | 경계 ↔ 개방 | 라벤더/모브 |

- **hue = 축, 채도/명도 = 강도**. 베이지안 축 점수에서 **결정론적으로** 산출(설명 가능: "이 코랄은 네 불안 점수 때문").

### 6.2 연출

- 다층 **메시 그라데이션** 레이어, 시작은 무채색(점토색).
- 성향 드러날 때 해당 색 레이어 opacity 0→상승, 위치 부드럽게 안착(확 번짐 X, 스밈 O). 깊은 blur로 경계 없이 섞임 → 나만의 오로라.
- 트윈 점토도 같은 팔레트를 먹음. `w_display`로 추종(§5.3).
- 딥 미드나잇 인디고(#0A0D1A) 위 발광. 원색·형광 금지.

### 6.3 자산화

내 6색 = 앱 테마 / 트윈 영구색 / SNS 공유 카드 배경 / 커플 합본('우리 색'). 재인터뷰 시 무채색 리셋 후 재충전.

## 7. 질문 세트

> 운영 원칙: 모든 질문은 **장면(상황)으로** 묻는다(추상 가치관 직접 질문 금지). 응답에서 감정 단어가 나오면 **꼬리 질문**으로 파고든다. 각 응답에 우도 벡터(§3.3)를 라벨링한다.
> 아래는 **시드(seed) 질문**이며, 운영자가 실사용자 인터뷰로 강화·교체할 기반이다.

### 7.1 공통 질문 (Universal) — 10개

> 유형 무관, 연애 필수 축. 3막에 항상 일부 포함. 비교·일관성 기준선.

1. 연인의 답장이 평소보다 늦으면, 머릿속에 제일 먼저 드는 생각이 뭐야?
2. "사랑받고 있다"고 느끼는 순간은 주로 언제야? (구체적 장면으로)
3. 연인과 다퉜을 때, 너의 첫 반응은 어떤 편이야? (먼저 풀려 함 / 시간을 둠 / 회피)
4. 서운한 마음이 들 때, 바로 말하는 편이야 속에 담아두는 편이야?
5. 연인에게 애정을 표현하는 너만의 방식이 있어? (말 / 행동 / 선물 / 시간 / 스킨십)
6. 관계에서 가장 두려운 건 뭐야? (예: 멀어짐 / 구속 / 변심 / 권태)
7. 연인이 힘들어할 때, 너는 해결책을 주는 편이야 그냥 들어주는 편이야?
8. 혼자만의 시간과 함께하는 시간, 어느 쪽이 더 필요해?
9. 연인에게 약한 모습(불안·눈물·실수)을 보이는 건 너한테 어떤 느낌이야?
10. 지금 이 관계에서, 너 스스로 고치고 싶은 대화 습관이 있어?

### 7.2 에니어그램 9유형 변별 질문 — 유형별 5개

> 각 유형의 **핵심 동기·핵심 두려움**을 자극하는 장면 질문. 응답이 해당 유형 우도를 높임. 괄호 안은 변별 의도.

**1번 — 개혁가 (핵심 두려움: 결함·부패. 동기: 옳고 바름)**
1. 연인이 약속을 자주 어기면, 화보다 먼저 드는 감정이 뭐야? (옳음 위반에 대한 분노)
2. 관계에서 "이건 이래야 한다"는 너만의 원칙이 있어? (내적 기준)
3. 네가 실수했을 때, 스스로를 얼마나 오래 자책하는 편이야? (자기비판)
4. 연인의 어떤 점을 "고쳤으면" 하고 바란 적 있어? (개선 충동)
5. 화가 나도 꾹 참고 예의를 지키려는 편이야? (분노 억압)

**2번 — 조력자 (두려움: 사랑받을 가치 없음. 동기: 필요한 존재)**
1. 연인에게 아무것도 해주지 못하는 날, 어떤 기분이 들어? (무용함 두려움)
2. 내 필요보다 연인의 필요를 먼저 챙기는 편이야? (자기희생)
3. "고맙다"는 말을 들을 때 너는 어떤 기분이야? (인정 욕구)
4. 도와줬는데 알아주지 않으면 서운해지는 편이야? (보답 기대)
5. 연인에게 도움을 '요청'하는 건 너한테 쉬워, 어려워? (받기의 어려움)

**3번 — 성취가 (두려움: 가치 없음·초라함. 동기: 인정·성공)**
1. 연인 앞에서 초라해 보일까 봐 신경 쓰인 적 있어? (이미지 관리)
2. 연인이 너를 '능력 있는 사람'으로 봐주는 게 중요해? (성취 인정)
3. 관계가 남들 보기에 어떻게 비치는지 신경 쓰는 편이야? (외부 시선)
4. 감정을 드러내는 것보다 효율적으로 처리하는 게 편해? (감정 보류)
5. 연인에게 "잘 보이고 싶다"는 마음, 인정 때문이야 불안 때문이야? (3번 vs 6번 변별)

**4번 — 개인주의자 (두려움: 정체성 없음·평범함. 동기: 특별함·진정성)**
1. 남들과 다른, 나만의 특별함을 연인이 알아봐주길 바라? (고유성)
2. 관계에서 깊고 진한 감정을 더 원하는 편이야? (정서적 강렬함)
3. 가질 수 없거나 멀리 있는 것에 더 끌린 적 있어? (결핍의 미학)
4. 연인과 있어도 가끔 외롭거나 이해받지 못한다 느껴? (근원적 결핍감)
5. 너의 감정 기복을 연인이 감당하기 힘들어한 적 있어? (감정 진폭)

**5번 — 탐구자 (두려움: 무능·고갈. 동기: 이해·자족)**
1. 연인이 너의 사적 시간/공간을 침범하면 어떤 느낌이야? (사생활 방어)
2. 감정을 나누는 것보다 생각·정보를 나누는 게 편해? (정서 거리)
3. 에너지가 닳으면 혼자 충전해야 하는 편이야? (고갈 두려움)
4. 연인의 감정 요구가 과하다 느낀 적 있어? (요구 부담)
5. 관계에서도 일정한 '내 영역'이 꼭 필요해? (독립 경계)

**6번 — 충실가 (두려움: 안전·확신 상실. 동기: 안전·확신)**
1. 관계가 너무 좋을 때, 오히려 불안해진 적 있어? (안정의 역설)
2. 연인의 마음을 자꾸 확인하고 싶어질 때가 있어? (확신 추구)
3. 최악의 시나리오를 미리 상상하는 편이야? (위험 대비)
4. 연인을 믿고 싶은데 의심이 드는 그 사이에서 갈등한 적 있어? (신뢰 양가성)
5. 그 '잘 보이고 싶음'이 관계가 불안해서야, 인정받고 싶어서야? (6번 vs 3번 변별)

**7번 — 열정가 (두려움: 결핍·고통에 갇힘. 동기: 즐거움·자유)**
1. 관계가 무겁고 진지해지면 답답해지는 편이야? (구속 회피)
2. 힘든 얘기보다 즐거운 계획을 더 하고 싶어? (고통 회피)
3. 한 사람/한 관계에 매이는 게 가끔 두려워? (선택지 보존)
4. 연인과 새로운 경험·자극을 계속 찾는 편이야? (신선함 추구)
5. 부정적 감정이 들면 빨리 다른 데로 주의를 돌리는 편이야? (회피 기제)

**8번 — 도전자 (두려움: 통제당함·약해짐. 동기: 자기 보호·주도)**
1. 연인에게 약한 모습 보이는 거, 너한테 어때? (취약성 노출)
2. 관계에서 주도권을 쥐는 게 편해, 맡기는 게 편해? (통제)
3. 부당하다 느끼면 바로 맞서는 편이야? (직면·정의감)
4. 누가 너를 통제하려 들면 어떤 반응이 나와? (지배 거부)
5. '잘 보이고 싶다'보다 '휘둘리기 싫다'가 더 큰 편이야? (8번 vs 3번 변별)

**9번 — 평화주의자 (두려움: 단절·갈등. 동기: 평온·연결)**
1. 갈등이 생기면 그냥 내가 맞추고 넘어가는 편이야? (갈등 회피)
2. 내가 뭘 원하는지 잘 모를 때가 있어? (자기 우선순위 흐림)
3. 연인 의견에 일단 맞춰주다 나중에 지친 적 있어? (수동적 융합)
4. 큰 다툼보다 어색한 침묵이 더 견디기 힘들어? (단절 두려움)
5. 결정을 미루거나 회피하는 편이야? (행동 관성)

### 7.3 운영자용 라벨링 가이드 (실사용자 강화 시)

각 질문·응답에 다음을 부착하여 엔진에 주입:
```
{
  questionId, act: 1|2|3|4,
  targetTypes: [추론 대상 유형],
  role: 'universal'|'confirmatory'|'disconfirmatory',
  likelihood: { "응답패턴A": {1:0.2, 3:0.8, ...}, ... },  // §3.3
  followUpTriggers: ["감정단어 감지 시 꼬리질문"]
}
```
- 운영자(=직접 인터뷰)는 응답을 보고 1~2개 후보 유형을 머릿속 베이즈로 갱신, **"○번 결 같은데 맞아?"** 제안+확인으로 검증.
- 변별 질문(5번류)은 **인접 혼동쌍**(3↔6, 8↔3, 2↔9 등)을 동시에 가르도록 설계됨.

## 8. 결과 공개 (트윈의 첫 무대)

### 8.1 ① 완성 세리머니
점토→트윈, 눈 뜨고 내 말투 첫 인사 + 감정 색 입혀진 완성형 등장.

### 8.2 ② 성향 카드 스와이프 (트윈이 표정·몸짓으로 연기)
- 카드마다 트윈이 해당 성향을 **몸으로 연기**(회피=뒤로 물러남, 인정욕구=으쓱, 불안애착=갸웃). 텍스트 안 읽어도 직관.
- 순서: ① 종합 정체성 한 줄 → ② 개인 성향(동기·두려움) → ③ 연애 성향(애착·언어·갈등) → ④ 시그니처(내 말투 드립 TOP) → ⑤ 종합 한마디.
- **따뜻한 카피 강제:** "회피형"(❌) → "갈등 앞에서 한 발 물러나 생각을 고르는 사람"(⭕).
- **제안+확인 내장:** Margin 작은 카드는 단정 대신 "이건 긴가민가한데 맞아?" → 끄덕이면 그 자리서 확정(미세 보정, 재인터뷰 회피).

### 8.3 ③ SNS 공유 카드 (피날레)
- 9:16 인스타 스토리. 주인공 = **나의 트윈**(앱 로고는 워터마크). 사람은 광고는 안 퍼뜨려도 자기 정체성은 퍼뜨림.
- 감성 칭호: "ISTJ 3w4"(날것) → "#한발물러나_생각을_고르는_사람"(감성 라벨).
- 내 6색 오로라 배경 + 세상에 하나뿐인 내 트윈.
- 프라이버시 가드: 집계·감성 라벨만, 대화 원문·구체 약점 비노출. → 커플 Wrapped 합본으로 확장.

### 8.4 ④ "나의 연애 색" 해설 페이지 (Why My Aura)

> **목적:** 인터뷰가 산출한 6색이 *왜* 이렇게 나왔는지를, 딱딱한 데이터가 아니라 **편지처럼 감성적으로** 풀어주는 페이지. 결과의 신뢰(설명 가능성)와 애착(나만의 서사)을 동시에 잡는다.
> **접근 경로 2가지:** ① 인터뷰 직후 **상단에서 떨어지는 배너**로 최초 노출 ② 이후 **테마 탭에서 상시 재열람**(§Settings 연동).

#### 8.4.1 진입 — 배너 드롭 (인터뷰 직후, 1회성 등장)

- 완성 세리머니(§8.1) → 카드 스와이프(§8.2) 흐름이 끝난 직후, 화면 **상단에서 부드럽게 떨어지는(slide-down + settle) 배너** 등장.
- 배너 비주얼: 내 6색 오로라가 흐르는 가로형 글래스모피즘 바. 트윈이 배너 옆에 살짝 얹혀 있음.
- 배너 카피(예): `🎨 너의 연애는 이런 색으로 물들어 있어 — 내가 읽어줄게` + 우측 `[펼쳐보기 ▾]`
- 터치 시 풀스크린으로 해설 페이지(§8.4.3)가 위로 펼쳐짐.
- **모션:** Aura Bloom과 동기화 — 배너가 떨어질 때 6색이 화면 전체로 처음 스며드는 연출과 동시(테마 최초 점등 모먼트).

#### 8.4.2 상시 접근 — 테마 탭 (재열람)

- 설정 탭의 **"나의 연애 색"** 섹션(§Settings 데이터 센터 내)에서 **언제든 다시** 열람.
- 배너는 1회성(휘발)이지만, 해설 페이지 자체는 영구 보관 → "그때 내 색이 왜 이랬지?"를 항상 되짚을 수 있음.
- 재인터뷰로 색이 바뀌면 해설도 새 버전으로 갱신(이전 버전 보관 옵션 — 성장 기록).

#### 8.4.3 해설 페이지 구성 (스크롤 서사)

> **카피 원칙:** 분석 리포트체 금지. **트윈이 나에게 건네는 편지/내레이션** 톤. 데이터 용어(에니어 N번, 축 점수)는 뒤에 숨기고, 감정의 언어로 먼저 말한다. 모든 표현은 §0 따뜻한 카피 원칙 준수(상처 라벨 금지).

1. **표지 — 나의 오라**
   - 내 6색이 합성된 오로라 풀스크린 + 감성 칭호(예: `#한발물러나_생각을_고르는_사람`).
   - 트윈의 한마디: "이게 너의 색이야. 왜 이렇게 나왔는지 들려줄게."

2. **색 하나씩 — 6개의 작은 이야기**
   - 6색 각각을 카드로. 각 카드 = **[색 스와치] + [감성 이름] + [왜 이 색인지 1~2문장] + [근거 한 줄]**.
   - 구조: *감성 설명을 먼저, 데이터 근거는 작게 뒤에.*
   - 예시(애착 축이 '불안' 쪽 코랄로 나온 경우):
     > 🪸 **코랄** — "너는 사랑 앞에서 자주 두근거리고, 가끔 그 마음이 너무 커서 불안해지기도 해. 그건 네가 이 관계를 그만큼 소중히 여긴다는 뜻이야."
     > *(작은 글씨) 이 색은 네 '애착 안정성' 이야기에서 왔어 — 답장이 늦을 때 먼저 떠오른 생각, 기억나?*
   - 예시(갈등 축이 '회피' 쪽 흐린 톤):
     > 🌫️ **흐린 세이지** — "너는 부딪히기 전에 한 발 물러나 마음을 고르는 사람이야. 성급하지 않아서, 네 말은 늦더라도 단단해."
   - **연출:** 각 색 카드가 보일 때 배경 오라에서 *그 색만* 잠깐 짙어졌다 가라앉음(어떤 색을 말하는지 시각 동기화). 트윈도 그 색으로 물듦.

3. **색의 조합 — 나라는 그림**
   - 6색이 어떻게 어우러져 "나"가 되는지 종합 내레이션 1문단.
   - 예: "코랄의 설렘과 흐린 세이지의 신중함이 섞여서, 너는 *깊이 사랑하지만 서두르지 않는* 사람이 됐어. 그게 너의 연애 색이야."
   - dominant 색(최강 축)을 "너를 가장 잘 설명하는 색"으로 짚어줌.

4. **닫는 말 — 트윈의 편지**
   - "이 색은 지금의 너야. 시간이 지나 네가 변하면, 나도 다시 너를 그릴게." (재인터뷰 §9로 자연 연결)
   - 하단 액션: `[💾 내 색 카드 저장]` `[📤 공유]`(§8.3 연동) `[🪞 다시 빚어보기]`(재인터뷰).

#### 8.4.4 데이터 ↔ 감성 매핑 규칙 (구현 필수)

해설 문구는 **즉흥 생성이 아니라**, 각 축의 점수 구간별로 사전 작성된 감성 카피 풀에서 선택·조합한다(신뢰성·안전성).

```
각 6축마다:
  scoreBand (예: 0–0.33 / 0.34–0.66 / 0.67–1.0) × 방향
   → 사전 큐레이션된 감성 카피 1개 매핑
  + 근거 한 줄: 해당 축을 가장 강하게 드러낸 인터뷰 응답/질문을 부드럽게 참조
     (원문 직접 인용 X, "답장 늦을 때 먼저 든 생각" 같은 맥락 환기만 — 프라이버시)
confidence(weight) 낮은 축 → 단정 카피 대신 "~한 결도 보여" 식 완화 표현
```

- **설명 가능성:** 모든 색은 베이지안 축 점수에서 결정론적으로 산출(UI 보강 명세 §1.2)되므로, 항상 "이 색은 네 ○○ 이야기에서 왔어"로 근거 제시 가능.
- **프라이버시:** 근거는 *맥락 환기*까지만. 실제 발화 원문·민감 내용은 노출 금지(온디바이스 마스킹).
- **따뜻함:** 부정적으로 들릴 축도 강점 프레임으로 번역("회피"→"신중", "불안"→"소중히 여김").

## 9. 재인터뷰 (Re-genesis)

- **말투 보존 분리:** `User_Persona_Matrix`(성향)만 점토로 리셋, `User_Tone_Vector`(말투)는 유지(카톡 기반이라 객관적).
- **사유 분기:** 되돌리기 시 트윈이 물음 — "결과가 안 맞아?(부정확 보정) / 내가 변한 것 같아?(성장 반영)". '그냥 맘에 안 들어'는 가벼운 마찰.
- **부드러운 쿨다운:** 무제한 금지. 트윈 의례적 되물음("우리 만든 지 얼마 안 됐는데… 정말 다시 빚을까? 너를 다시 알아가는 거 좋아").
- **연출:** 트윈이 점토로 풀어졌다 다시 빚어짐 = 거울을 다시 닦는 의식. 6색도 무채색 리셋 후 재충전.
- **남용 방지 근거:** 무제한 허용 시 '되고 싶은 나' 환상 → 거울이 거짓말 + 데이터 오염. 따뜻한 카피로 '맘에 안 듦' 충동 자체를 사전 차단.

## 10. 데이터 산출 & 적재

```
인터뷰 발화 ──┬─→ User_Persona_Matrix (에니어 P_enn, MBTI 보정, 애착, 방어기제, 6색 벡터)
              └─→ User_Tone_Vector 보강 (Chat_logic.md: 말투·드립·웃음·호흡)
정확도: 카톡만 업로드 = 50% → 인터뷰 완료 = 95% 해제(FUN-HOM 배너 소멸)
```
- 한 번의 인터뷰가 **두 자산 동시 적재**.
- 온디바이스 선처리(민감정보 마스킹, 연인 발화 무관). 결과는 개인 벡터 DB.
- **색 해설 자산(§8.4):** 6축 점수 → `AuraVector`(UI 보강 명세) + 축별 선택된 감성 카피 + 근거 맥락 참조를 묶어 `AuraStory` 객체로 저장. 테마 탭 상시 재열람·재인터뷰 시 갱신(이전 버전 보관 옵션).

### 11.3 구현·윤리 주의
- **STT 정확도가 우도 품질을 좌우** → confirm 되짚기 필수.
- 우도 테이블·affinity 행렬 품질이 추론 정확도의 전부 → 실사용자 인터뷰로 지속 강화(본 문서의 시드 질문 교체·증강).
- 짧은 인터뷰의 유형 추론은 본질적 확률 → **항상 제안+확인**, 단정 금지.
- 따뜻한 카피·웰빙 우선·재인터뷰 가드로 자기혐오/환상 양극단 방지.
- **색 해설(§8.4)은 즉흥 생성 금지** → 사전 큐레이션 카피 풀에서 선택·조합(신뢰성·안전). 근거는 맥락 환기까지만, 발화 원문 노출 금지.
- 실시간 3D·메시 그라데이션은 저사양 기기 성능 고려 → MVP 단계 전환·에셋 최적화.
